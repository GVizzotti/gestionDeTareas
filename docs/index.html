Sistema de Gestión de Tareas con API y Base de Datos

Documentación oficial del proyecto final. Este proyecto demuestra la implementación de una API REST con Flask, gestión de usuarios, autenticación usando bcrypt y una base de datos SQLite.

1. Instalación

Para configurar el proyecto en tu máquina local, sigue estos sencillos pasos. Asegúrate de tener Python 3 y pip instalados.

  1. Clona o descarga el repositorio desde GitHub.
  2. Abre una terminal en la carpeta del proyecto.
  3. Instala todas las dependencias necesarias con un solo comando:
pip install -r requirements.txt

2. Ejecución

Para correr la aplicación, necesitarás dos terminales abiertas.

Terminal 1: Iniciar el Servidor

En esta terminal, inicia la API de Flask. El servidor se encargará de gestionar los datos y la lógica.

python servidor.py

Terminal 2: Ejecutar el Cliente

En la segunda terminal, ejecuta el cliente de consola para interactuar con el sistema.

python cliente.py

3. Demostración y Pruebas

A continuación se muestra el flujo de uso completo del cliente de consola, desde el registro hasta el cierre de sesión.

Paso 1: Registro de un Nuevo Usuario

El primer paso es crear una cuenta. El cliente solicita un nombre de usuario y una contraseña.

Paso 2: Inicio de Sesión

Una vez registrado, el usuario puede iniciar sesión con sus credenciales.

Paso 3: Ver Tareas (Inicialmente Vacías)

Tras iniciar sesión, el usuario puede consultar sus tareas. Si es un usuario nuevo, la lista aparecerá vacía.

Paso 4: Crear una Nueva Tarea

El usuario puede añadir una nueva tarea proporcionando un título y una descripción opcional.

Paso 5: Ver Tareas Actualizadas

Después de crear una tarea, al volver a consultar la lista, esta aparecerá actualizada.

Paso 6: Cerrar Sesión

Finalmente, el usuario puede cerrar su sesión de forma segura, volviendo al menú principal.

4. Respuestas Conceptuales

¿Por qué es crucial hashear las contraseñas?

Hashear contraseñas impide que sean almacenadas en texto plano. Esto protege los datos del usuario si la base de datos es comprometida. Los hashes no son reversibles, lo cual evita que alguien pueda ver la contraseña original. En lugar de almacenar las contraseñas en texto plano, se transforman en una cadena de caracteres única e irreversible mediante un algoritmo como bcrypt, garantizando así la protección de la información del usuario.

Ventajas de usar SQLite en este proyecto

SQLite es una opción ideal para proyectos pequeños como este, ya que no requiere la configuración de un servidor y almacena los datos en un archivo local. Es una solución simple y eficiente, adecuada para un sistema básico de gestión de tareas. Al no necesitar un servidor de base de datos separado, consume muy pocos recursos del sistema, lo que la convierte en una excelente opción para el desarrollo local. Además, al estar encapsulada en un solo archivo, es increíblemente fácil de copiar, compartir y respaldar. En resumen, el proyecto completo es autocontenido y accesible.
